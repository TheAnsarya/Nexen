# 2026-05-15 Session 07

## Summary

Applied a central vcpkg include-path fallback for all native MSBuild projects after repeated Windows failures showed missing third-party headers from `Core.vcxproj` (not only `Utilities.vcxproj`).

## Failure Evidence

Run `25916829624` still failed in Windows jobs during InteropDLL target while compiling `Core.vcxproj` sources, with missing headers including:

- `magic_enum/magic_enum.hpp`
- `miniz/miniz.h`
- `lua.hpp`
- `kissfft/kissfft.hh`

## Root Cause

vcpkg include wiring was project-local and did not reliably propagate to all native project compile contexts in the InteropDLL target graph.

## Changes Made

- Updated `Directory.Build.props`:
	- Added shared fallback assignment for `VcpkgInstalledDir` when unset and local `vcpkg_installed` exists.
	- Added shared computed properties for triplet/install/include paths.
	- Added vcxproj-wide `ClCompile.AdditionalIncludeDirectories` injection for computed vcpkg include directory.
