## Technical Details

### Bug 1: Silent Exit When Already Running

When Nexen was already running and a user launched a second instance, `Program.cs` would silently log "Another instance is already running" and exit with no user feedback. Fixed by starting an Avalonia app with a custom message box dialog offering two choices:

- **Close and Restart Nexen** — Kills existing Nexen process(es) and launches a new instance
- **Leave Current Version Running** — Exits the second instance gracefully

### Bug 2: Unreliable Setup Wizard Restart

After the setup wizard completed, `Process.Start(Program.ExePath)` was called with no `ProcessStartInfo` and no error handling. Fixed by using `ProcessStartInfo` with `UseShellExecute = true` and wrapping in try/catch.
