# 2026-04-14 Session 10

## Summary

Fixed input controller configuration lockout reported in #1286 and applied parity hardening across other fixed-controller system tabs.

Main issue: setup commands in handheld/fixed-controller tabs were gated by persisted controller type. If config drifted to `None` or another invalid value, setup buttons became unavailable, blocking recovery and making input effectively unusable.

## Issues

- Fixed: #1286 (Can't config gameboy system controller)

## What Changed

1. Enforced canonical controller type on apply for fixed-controller systems:
	- Game Boy now forces `ControllerType.GameboyController` in `ApplyConfig()`.
	- GBA now forces `ControllerType.GbaController` in `ApplyConfig()`.

2. Removed setup-button lockout for fixed-controller tabs:
	- Game Boy, GBA, Lynx, and WonderSwan setup commands no longer depend on persisted `Config.Controller*.Type` for command enablement.
	- Setup commands now always remain available for these fixed-controller tabs.

3. Normalized setup dialog config type before editing:
	- Before opening `ControllerConfigWindow`, cloned config type is set to the canonical type for each tab/system.
