## Work Done

### Research

- Analyzed all SNES hot paths: CPU dispatch, memory manager, PPU, coprocessor sync
- Identified `(this->*_execRead)()` / `(this->*_execWrite)()` in `SnesMemoryManager::Read()`/`Write()` as the primary optimization target — member function pointer dispatch on every memory operation blocks compiler inlining
- Confirmed PPU, debugger hooks, coprocessor sync, and notification system are already well-optimized

### Implementation

1. **Memory dispatch optimization (#1362)**: Replaced member function pointer calls with `ExecReadTiming()` / `ExecWriteTiming()` inline methods using a switch on `_cpuSpeed` (3 values: 6, 8, 12). Enables compiler inlining of `IncMasterClock<N>()` templates at each call site. Added `[[likely]]` on speed=8 (most common case).

2. **CPU method inlining (#1363)**: Added `__forceinline` to `CheckFlag()`, `SetZeroNegativeFlags()` (both overloads), `ClearFlags()`, `SetFlags()` — all simple 1-3 line methods called millions of times per frame.

3. **`[[nodiscard]]` attributes (#1364)**: Added to `CheckFlag()`, `GetOpCode()`, `GetByteValue()`, `GetWordValue()` — pure query methods.

4. **Memory dispatch benchmark (#1365)**: Created `SnesMemoryManagerBench.cpp` with function pointer vs switch dispatch comparisons, mixed-speed tests, and simulated read path benchmark.
