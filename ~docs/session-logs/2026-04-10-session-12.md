## Release Notes

The v1.4.25 release was published manually after a series of CI failures:

1. **First build run 24262580696** — all Linux/macOS builds failed (`clock_cast` C++23 issue)
2. Fixed in `d87e17c9`, retriggered as **run 24263305790**
3. All 10 platform build jobs passed in run 24263305790
4. **Release job failed with 403** — softprops tried to CREATE (not update) and GitHub rejected it
   - Root cause: deleting and recreating the git tag broke GitHub's tag-release association permanently
   - `GET /releases/tags/v1.4.25` returned 404 even after PATCH/re-create attempts
   - `gh release create v1.4.25` also returned `untagged-...` URL (GitHub tombstone issue)
5. **Manual artifact upload** — downloaded all 10 CI artifacts, repackaged Linux binaries as tarballs
   with Python-fixed permissions (0755), uploaded all to the release via `gh release upload`
6. Published with `gh release edit v1.4.25 --draft=false`
7. **Workflow fix** — added `actions/checkout` and `body_path: .github/release-notes/latest.md`
   to the release job so future releases avoid the encoding issues and have proper description

**Key lesson**: Never delete a git tag that has a GitHub release associated. Instead, use the
GitHub API to update the release's `target_commitish` directly, or delete-release first, then
delete-tag, then recreate-tag, then create-release in that exact order.
