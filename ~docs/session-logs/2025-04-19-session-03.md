## Work Done

### Epic #1381 — SNES Performance Optimization Phase 4

Researched and implemented 4 PPU rendering optimizations:

1. **#1382 ApplyColorMath window mask pointer hoist** — Hoisted `_windowMask[SnesPpu::ColorWindowIndex]` to a local `const bool*` pointer before the 4 loop variants. Eliminates double array dereference per pixel (up to 256/512 per scanline).

2. **#1383 PrecomputeWindowMasks MaskLogic switch hoist** — Moved `_state.MaskLogic[layer]` switch statement outside the inner x loop for the 2-window case. Creates 4 separate loops (Or/And/Xor/Xnor) that each run without per-pixel branching. MaskLogic is constant per layer per scanline.

3. **#1384 ConvertToHiRes pointer arithmetic** — Replaced indexed access (`src[x]`, `dst[x << 1]`) with sequential `*src++`/`*dst++` pointer arithmetic. Cleaner codegen, avoids shift per iteration.

4. **#1385 ApplyColorMath [[likely]] non-hiRes path** — Added `[[likely]]` attribute to the non-hiRes else branch. Most SNES games use standard 256-pixel resolution.

### Build & Test Results

- **Build:** Successful (Release x64)
- **Tests:** 3690/3690 passed (24050ms)
- **Benchmarks:** No regressions. Optimizations target combined rendering path overhead not fully captured by isolated micro-benchmarks.
