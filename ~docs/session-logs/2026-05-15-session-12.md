# 2026-05-15 Session 12

## Summary

Fixed current Windows CI command-line failure (`MSB1006`) and introduced a Windows-only workflow mode to iterate quickly without rerunning Linux/macOS/AppImage during triage.

## Root Cause

Windows InteropDLL step passed an `IncludePath` value via `/p:IncludePath=...;...` on msbuild command line.
MSBuild interpreted semicolon-delimited entries as separate property switches and failed with:

- `MSB1006: Property is not valid`
- `Switch: D:\a\Nexen\Nexen\vcpkg\installed\x64-windows\include`

## Changes Made

- Updated `.github/workflows/build.yml`:
	- Added `workflow_dispatch` input:
		- `windows_only` (boolean, default `true`)
	- Removed invalid Windows InteropDLL switch:
		- dropped `/p:IncludePath=...`
	- Added skip conditions for non-Windows jobs when `windows_only=true`:
		- `linux`
		- `appimage`
		- `macos`

## Local Validation

Strict pre-CI local check using the exact Windows InteropDLL msbuild command now in workflow:

- Result: `Build succeeded`
- Exit code: `MSBUILD_EXIT=0`
