## Work Done

### xunit v2 → v3 Migration (#1202)

- Replaced `xunit` 2.9.3 → `xunit.v3` 3.2.2 in both test projects
- Added `<OutputType>Exe</OutputType>` per v3 requirements (test projects are now standalone executables)
- Removed `Xunit.SkippableFact` 1.5.61 from MovieConverter.Tests
- Replaced `[SkippableFact]` → `[Fact]` and `Skip.IfNot()` → `Assert.SkipUnless()` (23 instances in RealFileConversionTests.cs)
- Added `ReactiveUI` 23.1.8 direct package reference to Nexen.Tests
- Created `Tests/TestInitializer.cs` — `[ModuleInitializer]` that calls `RxAppBuilder.CreateReactiveUIBuilder().WithCoreServices().BuildApp()` to initialize ReactiveUI for the test context
- Fixed pre-existing `SaveStateOrigin_HasFourMembers` test (enum now has 5 members after Designated was added)

### Key Discovery

- ReactiveUI 23.x (used via ReactiveUI.Avalonia 11.4.12) requires explicit initialization via builder pattern
- In xunit v2, tests that construct ViewModels worked because the test runner environment provided sufficient context
- In xunit v3 (standalone executable), ReactiveUI initialization must be explicit — solved with `[ModuleInitializer]`
- This also fixed 369 previously-silent test failures in Nexen.Tests that weren't being discovered/counted in prior sessions
