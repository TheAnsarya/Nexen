## Code Changes

- Added `GenesisVdp::ReadPortByte(uint32_t addr)` with byte-accurate behavior:
- Data port high-byte reads now return the current buffer high byte without priming.
- Data port low-byte reads now return low byte and then prime the next buffer.
- Control/status byte reads now use a status latch so even/odd reads share one status sample and one clear-on-read side effect.
- HV counter byte reads now return the corresponding high/low byte directly.
- Added status latch state fields and serialization for byte-read latch consistency.
- Invalidated status latch on non-split reads and VDP writes.
- Updated MMU VDP read8 path to call `ReadPortByte()` instead of constructing byte reads from `ReadVdpPort()` word reads.
- Suppressed frame-0 `VDP_CTRL_R` startup trace logging to reduce synthetic first-diff noise.
