# 2026-05-15 Session 06

## Summary

Resolved persistent Windows InteropDLL include failures by correcting manifest install-root usage in the Windows vcpkg setup/build path wiring.

## Failure Context

Latest run `25916112394` failed only on Windows matrix jobs at `Build C++ core (InteropDLL)` with:

- `miniz/miniz.h` not found
- `magic_enum/magic_enum.hpp` not found

## Root Cause

Windows setup used manifest mode but wrote/consumed `VcpkgInstalledDir` as `vcpkg\installed`, while manifest installs were being resolved under workspace `vcpkg_installed` semantics.

## Changes Made

- Updated `.github/workflows/build.yml` (Windows job):
	- Added explicit `--x-install-root="${{ github.workspace }}\vcpkg_installed"` to vcpkg install command.
	- Updated exported `VCPKG_INSTALLED_DIR` to `${{ github.workspace }}\vcpkg_installed`.
	- Updated InteropDLL msbuild property `VcpkgInstalledDir` to `${{ github.workspace }}\vcpkg_installed`.
