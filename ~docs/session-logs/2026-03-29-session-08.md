# Session Log: 2026-03-29 Session 08

## Summary

Implemented CI policy change to stop automatic Build workflow spam and require explicit release intent.

## Issue Worked

- #1101 Build Nexen manual-only policy and serialized execution

## Workflow Changes

Updated `.github/workflows/build.yml`:

1. Trigger policy

- Removed push/tag auto-trigger.
- Added `workflow_dispatch` inputs only.

2. Concurrency policy

- Added top-level concurrency group: `build-nexen-single`.
- Enabled `cancel-in-progress: true` to prevent multiple simultaneous runs.

3. Release policy

- Release job now runs only when manual input `run_release=true`.
- Added `release_tag` input validation (`vX.Y.Z` required when release requested).
- Release action now uses explicit manual tag input (`tag_name`, `name`, `target_commitish`).

## Operational Outcome

- No automatic Build workflow on every commit.
- Release creation occurs only when explicitly requested.
- Workflow execution is serialized to one active run.

## Notes

This aligns CI behavior with explicit operator control and reduces accidental multi-run noise in GitHub Actions.
