# 2026-05-15 Session 09

## Summary

Added explicit vcpkg include wiring to `Core.vcxproj` after repeated Windows InteropDLL failures showed missing third-party headers originating from Core compile units.

## Why This Change

Multiple runs still failed on Windows with `Build C++ core (InteropDLL)` despite workflow/path fixes. Error origins were dominated by `Core.vcxproj` compile units (not only Utilities), indicating the Core project itself needed explicit vcpkg include visibility.

## Changes Made

- Updated `Core/Core.vcxproj`:
	- Added vcpkg property computation:
		- `NexenVcpkgTriplet`
		- `NexenVcpkgInstalledRoot`
		- `NexenVcpkgIncludeDir`
	- Updated all configuration `ClCompile.AdditionalIncludeDirectories` entries to include:
		- `../;$(NexenVcpkgIncludeDir);%(AdditionalIncludeDirectories)`
