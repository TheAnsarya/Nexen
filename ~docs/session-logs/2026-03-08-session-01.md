## Work Completed

### Audit

- Explored full TAS architecture via search across C++ and C# codebase
- Identified 32 issues across 7 files (11 HIGH, 16 MEDIUM, 5 LOW)
- Mapped all movie-related files: MovieManager, MovieRecorder, NexenMovie, RewindManager, RewindData (C++); MovieData, InputFrame, ControllerInput, 7 format converters (C#); PianoRollControl, TasEditorWindow (Avalonia UI)

### C++ Fixes

1. **MovieRecorder.cpp (#603):** Replaced `_inputData << ("|" + device->GetTextState())` with `_inputData << '|' << device->GetTextState()` in both `RecordInput()` and `CreateMovie()`. Eliminates temporary `std::string` allocation per device per frame at 60 FPS.

2. **NexenMovie.cpp (#604 Phase 1):** Added `reserve()` with line-counting pre-scan before `push_back` loop in `Play()`. Avoids repeated vector reallocations for large movies (200K+ frames). Phase 2 (data structure redesign to eliminate per-frame string parsing) deferred.

### C# Fixes

3. **PianoRollControl.cs (#606):** Changed `_buttonLabelCache` to use `Clear()` instead of `new Dictionary<>` on zoom changes. Changed `_frameNumberCache` to use sliding window eviction (remove entries outside visible range) instead of full clear.

### Tests (#607)

- Created `TasPerformanceTests.cs` with 24 new tests:
	- TruncateAt stress tests (10K frames, 100K frames, edge cases, data preservation)
	- Clone stress tests (various sizes, independence verification)
	- CRC32 determinism and uniqueness tests
	- InputFrame parsing round-trip tests (NexenLogLine single/bulk, special lines)
	- ControllerInput format tests (SmvFormat round-trip, empty input, clone)

- Added 3 new benchmarks to `MovieDataBenchmarks.cs`:
	- `TruncateAt_Middle` — Clone + truncate at midpoint
	- `CalculateInputCrc32` — CRC32 over all frames
	- `NexenLogLine_RoundTrip` — serialize → deserialize → serialize cycle
