# Session Log - 2026-04-30 (Session 07)

## Summary

Fixed a startup routing bug where `.rgd` recent-game session files were incorrectly treated as ROM files when opened through `LoadRomHelper.LoadFile` (for example via command-line startup). This caused `EmuApi.LoadRom` to fail with `GameLoadFailed`, which could leave the Genesis diagnostics in a misleading zero-state context.

## Work Completed

- Reproduced the failure by launching `Nexen.exe` with a `.rgd` path argument.
- Confirmed in logs that `.rgd` was routed to `LoadRom` instead of `LoadRecentGame`.
- Patched `LoadRomHelper.LoadFile` to detect `.rgd` and call `LoadRecentGame(filename, false)`.
- Rebuilt `Release x64` successfully.
- Re-ran `.rgd` launch validation and confirmed routing now uses `LoadRecentGame` with `GameLoaded` notification.
