## Key Performance Findings

### std::format vs String Concatenation on MSVC

| Pattern | Relative Speed |
|---------|---------------|
| std::format vs simple `+` concat | **2–3x SLOWER** (don't use!) |
| std::format vs stringstream | **3.8x FASTER** (always use!) |
| std::format vs snprintf | ~Similar (marginal at best) |

**Rule**: Only use `std::format` to replace `std::stringstream`/`std::ostringstream`. Keep simple `operator+` for short string building.

### Copy Elimination Impact

| Pattern | Speedup |
|---------|---------|
| vector<string> by-value → const ref | **8.8x** |
| unordered_set<string> by-value → const ref | **66.3x** |
| vector<uint8_t> by-value → const ref | ~7% |
| string → string_view | ~16% |
