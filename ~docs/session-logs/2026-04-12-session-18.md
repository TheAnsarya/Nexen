## Changes Made

### #1232 — Fix async void non-handlers (2a7b7aff)

Changed 8 `async void` methods to `async Task`:

- `CommentEditWindow.EditComment()` — added `using System.Threading.Tasks`
- `LoadRomHelper.LoadRom()` and `LoadPatchFile()` — internal calls now use `await`
- `RomTestHelper.RunTest()`
- `ShortcutHandler.Reset()`, `PowerCycle()`, `ReloadRom()`, `PowerOff()`

Updated all callers with `_ =` (explicit discard) in sync contexts (OnClick lambdas, switch cases) or `await` in async contexts (LoadPatchFile calling LoadRom).

**Files changed:** CommentEditWindow.axaml.cs, LoadRomHelper.cs, RomTestHelper.cs, ShortcutHandler.cs, DisassemblyView.axaml.cs, MainMenuViewModel.cs, MainWindow.axaml.cs, DebuggerWindowViewModel.cs

### #1233 — Fix ToolRefreshHelper lock pattern (822e0a6b)

- Replaced `lock(callback)` with the already-declared `lockObject` variable (was unused — a bug)
- Moved `done = true` before `callback()` to prevent double-entry on exception
- Added `try-catch` with `Log.Error()` around both callback invocations
- Added `using Nexen.Utilities` for Log access

**Files changed:** ToolRefreshHelper.cs

### #1234 — Add [[nodiscard]] to C++ getters (88db4c90)

Added `[[nodiscard]]` to 19 methods across 7 headers:

- `BreakpointManager.h` — HasBreakpoints()
- `ScriptManager.h` — HasScript()
- `IDebugger.h` — CheckStepBack(), IsStepBack(), GetStepBackConfig(), GetFrozenAddressManager(), GetSupportedFeatures(), GetCpuCycleCount(), GetCpuFlags(), GetBreakpointManager(), GetCallstackManager(), GetAssembler(), GetEventManager(), GetTraceLogger(), GetPpuTools()
- `TraceLogFileSaver.h` — IsEnabled()
- `SmsVdp.h` — GetVisiblePixelIndex()
- `WsTypes.h` — WsFlagRegister::Get()
- `WsPpu.h` — GetBgColor()

Note: 99 override declarations in 10 platform debuggers (NES, SNES, GB, PCE, SMS, WS, Lynx, GBA, Atari2600, ChannelF) don't need `[[nodiscard]]` — MSVC propagates from base virtual methods.
