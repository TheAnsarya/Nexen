## Benchmarks Added Per Issue

### SNES PPU Benchmarks (Issue: None — new coverage)

`SnesPpuBench.cpp` covers:

- 4bpp tile pixel extraction: single pixel, full 8-pixel row, 256-pixel scanline
- 2bpp tile pixel extraction: 256-pixel scanline (Mode 0)
- CGRAM palette lookup: sequential, pseudorandom (realistic) scanline
- Color math (add/subtract/halved): single pixel, full scanline, windowed scanline
- Mode 7 affine transform: step accumulation, full 256-pixel scanline with VRAM
- Window mask precompute: per-scanline cost, per-pixel lookup, 6-layer full-scanline
- Per-scanline settings fetch: non-inline call vs cached copy, per-scanline, full-frame
- BG priority resolution: 5-layer scanline priority selection

### Genesis VDP Benchmarks (Issue: None — new coverage)

`GenesisVdpBench.cpp` covers:

- 4bpp tile pixel extraction: single pixel, unrolled row, h-flip row, H40 scanline, H32 scanline
- CRAM color conversion: single entry, all 64 entries, 320-pixel scanline lookup
- DMA fill: 1KB, 16KB, memset baseline for comparison
- DMA VRAM-to-VRAM copy: 1KB, 16KB, memcpy baseline for comparison
- Sprite attribute table scan: max 80 sprites (H40), few visible (early exit)
- Full background scanline render: H40 (320 pixels, nametable + tile + CRAM)
- Real GenesisVdp object: `Run()` one scanline, DMA fill via `WriteControlPort()`

### Debugger Refresh Benchmarks (Issue #749)

`DebuggerRefreshBench.cpp` covers:

- Disassembly cache lookup: NES (32KB), SNES (512KB), random access
- `GetNthSegment` comment extraction: single-line, multi-line line 2, full window pass
- `HexUtilities::ToHex` address formatting: 16-bit, 24-bit, 3-byte opcodes, full window
- Label/comment hash map lookup: 100% hit rate, sparse miss, sorted-vector comparison
- `CodeLineData` struct fill + memcpy: single row, full 64-row window
- Access counter array lookup: 64-row window
- Full window refresh simulation: all steps combined for 64 rows

### NES PPU Extensions (PpuRenderBench.cpp)

Sprite evaluation:

- `BM_NesPpu_SpriteEval_NoneVisible`: no sprites on scanline (idle cost)
- `BM_NesPpu_SpriteEval_8Visible`: 8 sprites on scanline (full OAM fill)
- `BM_NesPpu_SpriteEval_MaxDensity`: 64 sprites evenly spread (worst case)
- `BM_NesPpu_SpriteEval_LargeSprites_8x16`: 8×16 sprite range check

Pixel priority resolution:

- `BM_NesPpu_PixelColor_BgOnly`: BG-only fast path
- `BM_NesPpu_PixelColor_8Sprites_PriorityResolution`: per-pixel with 8 sprites
- `BM_NesPpu_PixelColor_FullScanline_8Sprites`: full 256-pixel scanline

### GBA DMA Benchmarks (Issue: None — new coverage)

`GbaDmaBench.cpp` covers:

- Halfword DMA copy: 16B, 256B, 1KB, 16KB
- Word DMA copy: 16B, 256B, 1KB, 16KB
- memcpy baseline for each size (theoretical floor)
- DMA channel priority scan: none active, channel 3 active, channel 0 active
- Audio FIFO DMA: 4-word (16-byte) repeat transfer
- DMA enable/disable state machine cycle
- HBlank trigger check (4 channels)
- Address mode dispatch: Increment, Decrement, Fixed, IncrementReload

### Save State Compression Benchmarks (Issue #422 context)

`SaveStateCompressionBench.cpp` covers:

- miniz level 1 (MZ_BEST_SPEED): 100KB compressible, 100KB random, 400KB compressible, 400KB mixed
- miniz level 6 (MZ_DEFAULT_LEVEL): 100KB compressible, 400KB mixed
- miniz level 0 (no compression): 100KB
- memcpy baseline: 100KB and 400KB
- Decompression: 100KB compressible, 100KB random, 400KB mixed
- Ratio profile: all levels 0/1/3/6/9 on 200KB mixed payload

### Lua Hook Extensions (LuaHookBench.cpp)

- `BM_LuaHook_PerInstruction_Count1`: per-instruction hook (worst case overhead)
- `BM_LuaHook_PerInstruction_Count10`: throttled hook (every 10 instructions)
- `BM_LuaHook_PerInstruction_Count100`: coarser throttling
- `BM_LuaHook_NoHook_Baseline`: same script, no hook (comparison baseline)
- `BM_LuaHook_EmulatorGuard_NoScript`: `if (hasHooks)` branch check with no script loaded
- `BM_LuaHook_EmulatorGuard_WithScript`: same check with script loaded
