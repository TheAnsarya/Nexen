## Bug Details

### 1. Recent Play Timing (#1187)

- `_lastRecentPlayTime = 0` (Unix epoch) caused `ShouldSaveRecentPlay()` to return true immediately on ROM load
- `ResetRecentPlayTimer()` existed but was never called
- Fix: Initialize to `std::time(nullptr)` + call `ResetRecentPlayTimer()` in `GameLoaded` handler

### 2. Timestamp DST Offset (#1188)

- `ParseTimestampFromFilename()` zero-initialized `tm_isdst = 0`, telling `mktime()` DST was not in effect
- During DST months, timestamps showed 1 hour ahead
- Fix: Set `tm.tm_isdst = -1` for auto-detection

### 3. Screen Not Updating After Paused State Load (#1188)

- `UpdateFrame` was only called when `_emu->IsPaused()` was true
- Fix: Always push saved frame after successful state load (skip only during AVI recording)

### 4. Pause Icon Rendering (#1188)

- Unicode `⏸` (U+23F8) rendered inconsistently as a blue square with white bars
- Fix: Replaced with two drawn `Border` rectangles (3×10px, black, in a horizontal StackPanel)
