# Session Log — 2026-04-05 Session 03

## Summary

Investigated and fixed startup crash issues reported in #1076 (Windows TypeInitializationException) and #1039 (Linux NexenCore.dll missing / premature `~/.config/Nexen/` creation).

## Root Cause Analysis

### #1076 — TypeInitializationException on Windows

`SetupWizardWindow` inherits from `NexenWindow`, whose `OnInitialized()` calls `SetTextRenderingMode()` which accesses `ConfigManager.Config.Preferences.FontAntialiasing`. This triggers the full `HomeFolder` → `LoadConfig()` chain before the user's environment is ready. If any step fails (permissions, corrupted config, missing special folders), it wraps as a `TypeInitializationException`.

### #1039 — Linux NexenCore.so Not Found / Premature HomeFolder Creation

Same root cause for premature directory creation — `SetupWizardWindow : NexenWindow` triggers `ConfigManager.Config` access during setup wizard, creating `~/.config/Nexen/` before the user has chosen their preferred location.

For the native library issue, the `DllImportResolver` silently returned `IntPtr.Zero` when the library wasn't found, giving users the unhelpful "Unable to load shared library" error with no diagnostic info about what paths were searched.

## Changes Made

### UI/NexenWindow.cs

- `SetTextRenderingMode()` now checks `App.ShowConfigWindow` and skips `ConfigManager.Config` access during setup wizard mode
- Added try-catch fallback to `SubPixelAntialias` default for resilience
- Removed stray semicolon after switch block
- Added `using Nexen.Utilities;` for `Log` class

### UI/App.axaml.cs

- `Initialize()` — Refactored theme selection to guard `ConfigManager.Config` access with try-catch
- Defaults to `ThemeVariant.Light` on failure instead of crashing

### UI/Program.cs

- Added dedicated `TypeInitializationException` catch handler that logs inner exception type name and stack trace
- `DllImportResolver` — Skips `HomeFolder` fallback when `App.ShowConfigWindow` is true (prevents premature `~/.config/Nexen/` creation)
- `DllImportResolver` — Added diagnostic logging when native library resolution fails (shows original name, resolved name, searched paths)
- Post-wizard config check — Replaced `ConfigManager.GetConfigFile()` with direct file path checks to avoid triggering `HomeFolder` auto-detection on wizard cancel
