# Session Log - 2026-04-29 (Session 59)

## Summary

Implemented centralized routing for ambiguous .bin ROM files so loader selection is content-aware instead of relying only on probe order. This prevents Genesis ROMs from being misrouted to Channel F when both consoles advertise .bin support.

## Code Changes

- Added centralized .bin scoring helpers in [Core/Shared/Emulator.cpp](Core/Shared/Emulator.cpp):
	- Genesis score boosts for SEGA header marker at $0100 and larger ROM profiles
	- Channel F score boosts for valid Channel F size profile and penalties for Genesis marker
- Added an early, centralized .bin candidate pass in [Core/Shared/Emulator.cpp](Core/Shared/Emulator.cpp) before normal extension/signature fallback.
- Added additional Channel F guardrails in [Core/ChannelF/ChannelFConsole.cpp](Core/ChannelF/ChannelFConsole.cpp):

	- Reject SEGA-header images as UnknownType to allow Genesis fallback
