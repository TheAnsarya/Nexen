## Technical Notes

### V30MZ Encoding Limitation

Poppy's V30MZ instruction set implementation (`InstructionSetV30MZ.cs`) is skeletal:

- **Supported**: Implied ops (nop, hlt, cli, cld, sti, etc.), conditional jumps (jne, je, jb, etc.), loop instructions
- **NOT supported**: Any parametric instruction (mov, add, sub, cmp, push, pop, in, out, jmp near/far, call, ret with imm16, etc.)
- The `V30AddressingMode` enum and `EncodeModRM()` helper exist but have no codegen path
- The WS test ROM works around this using `.byte` directives for manual encoding

### Mnemonic-as-Label/Parameter Fix Summary

Three parser changes were made (across sessions):

1. **Mnemonic-as-label lookahead** (Parser main loop): If a Mnemonic token is followed by a Colon, parse as label via `ParseMnemonicLabel()` instead of `ParseInstruction()`
2. **`CheckNext(TokenType)`**: Helper to peek at next token type without advancing
3. **Macro parameter acceptance**: `ParseMacroDefinition()` now accepts both Identifier and Mnemonic tokens as parameter names
