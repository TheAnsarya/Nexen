## Changes Made

### C++ Core (`Core/Shared/Emulator.cpp`)

- Moved `SaveRecentGame` and `SaveRecentPlayState` calls BEFORE `TryLoadRom` in `InternalLoadRom()` — the root cause of the crash was that these state serialization calls happened after `_consoleMemory` was zeroed, causing corrupted saves and crashes on subsequent ROM loads
- `Stop()` now called with `preventRecentGameSave=true` to avoid double-save with invalid memory

### UI (`UI/ViewModels/RecentGamesViewModel.cs`)

- Game selection screen now shows recent ROMs from `RecentFiles` config in addition to `.rgd` session files
- Added `IsRecentRomFile` property to `RecentGameInfo` for direct ROM loading

### UI (`UI/Controls/StateGrid.axaml.cs`)

- `RecentGames` mode now uses full grid layout instead of shrinking to 1×1 for single entries
