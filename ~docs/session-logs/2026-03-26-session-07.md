## Changes

### 1. BaseVideoFilter::UpdateBufferSize() — Remove Redundant Lock

`SendFrame()` acquires `_frameLock` via `AcquireSafe()`, then calls `UpdateBufferSize()` which acquired `_frameLock` again (recursive entry). Removed the inner `Acquire()`/`Release()` since the caller always holds the lock. Also eliminates double-lock overhead on the `make_unique` heap allocation.

### 2. Cache GetEmulationSpeed() — 4 Sites

`GetEmulationSpeed()` checks 3 flag conditions + reads config value. Was called twice in single boolean expressions at:

- `Emulator::Run()` (line 134)
- `SnesPpu.cpp` (line 591)
- `GbaPpu.cpp` (line 163)
- `PceVpc.cpp` (line 195)

Cached to local `uint32_t emulationSpeed` variable (same pattern already used at `Emulator::GetFrameDelay()`).

### 3. GB PPU — [[likely]] on Visible Scanline Path

`GbPpu::ExecCycle()`: `if (_state.Scanline < 144)` is true 144/154 = 93.5% of PPU cycles. Added `[[likely]]` to help branch prediction and code layout on this per-cycle hot path (~4.2M calls/sec).

### 4. GBA PPU — [[unlikely]] on VBlank Early Exit

`GbaPpu::RenderScanline()`: `if (_state.Scanline >= 160 || _lastRenderCycle >= 1056)` is an early exit that fires during vblank/end-of-render. Most calls are during visible scanlines. Added `[[unlikely]]`.
