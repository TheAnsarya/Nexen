# Session Log: 2026-03-29 Session 09

## Summary

Targeted AppImage runtime crash hardening pass to address unresolved ICU startup failures reported in #1039.

## Issue Worked

- #1102 AppImage script native-runtime flag and clang flag alignment

## Root Cause

AppImage scripts overrode `PUBLISHFLAGS` with single-file publish settings that omitted:

- `-p:IncludeNativeLibrariesForSelfExtract=true`

That omission can break startup for app-local ICU/native dependencies in single-file AppImage outputs.

The scripts also used `NEXENFLAGS="-stdlib=libc++"`, which bypasses makefile internal optimization appends.

## Changes Applied

Updated both scripts:

- `Linux/appimage/appimage.sh`
- `Linux/appimage/appimage-arm64.sh`

Changes:

1. Added native extraction flag in `PUBLISHFLAGS`:

- `-p:IncludeNativeLibrariesForSelfExtract=true`

2. Switched clang flag injection to makefile extension variable:

- `NEXENFLAGS="-stdlib=libc++"` -> `EXTRA_CXXFLAGS="-stdlib=libc++"`

## Expected Outcome

- AppImage single-file publish includes/extracts native runtime assets consistently.
- AppImage clang builds preserve internal makefile optimization flags while still using libc++ for C++23 `<format>` support.
