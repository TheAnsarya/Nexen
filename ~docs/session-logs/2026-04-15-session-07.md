# 2026-04-15 Session 07

## Summary

Implemented issue #1290 by fixing menu/config auto-pause so it only triggers while a game is actively running.

This prevents ROM load/startup from being auto-paused when `Pause when in menu and config dialogs` is enabled.

## Root Cause

`timerUpdateBackgroundFlag(...)` evaluated menu/config pause conditions without requiring an active game state. During ROM load transitions this could pause emulation, leaving gameplay paused right after load.

## What Changed

In `UI/Windows/MainWindow.axaml.cs`:

1. Added `isGameRunning = EmuApi.IsRunning()` in the pause tick loop.
2. Replaced inline menu/config pause conditions with `ShouldPauseInMenusAndConfig(...)`.
3. Added `internal static` helper to centralize and test pause decision logic.

In tests:

1. Added `Tests/Windows/MainWindowPauseBehaviorTests.cs`.
2. Added five regression tests covering enabled/disabled and running/non-running combinations for menu/config contexts.
