### 3. Hot-Path Optimizations (#961)

Created issue #961 `[14.11] Atari 2600 TIA/Bus hot-path optimization and benchmark expansion`.

**Changes Applied:**

#### Atari2600Tia.h

- **Batch color clock stepping:** Replaced per-iteration loop in `StepColorClocks` with batch increment + single boundary check. Eliminates ~59,736 loop iterations per frame, replacing with N additions + 1 conditional.
- **Branch hints:** Added `[[unlikely]]` on WsyncHold (rare WSYNC), HmovePending (rare HMOVE), scanline boundary (262×/frame out of 59,736), and frame wrap (1×/262 scanlines). Added `[[unlikely]]` on zero-volume audio check.
- **Lambda extraction:** Moved `advanceChannel` and `toSignedSample` lambdas to static member functions (`AdvanceAudioChannel`, `ToSignedAudioSample`). Added `[[likely]]` on counter decrement, `[[unlikely]]` on counter reload.

#### Atari2600Bus.cpp

- **Cartridge read hint:** Added `[[likely]]` on cartridge ROM check ($1000-$1FFF) — most common read target during execution.

#### Atari2600Riot.h

- **Timer hints:** Added `[[likely]]` on divider counter > 0 (counting down), `[[unlikely]]` on divider reload (fires at divider rate), `[[unlikely]]` on timer underflow (rare).
