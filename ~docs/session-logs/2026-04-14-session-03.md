# Session Log — 2026-04-14 Session 03

## Summary

Continued issue #1076 with a whole-render-pipeline audit against ares, Mednafen, MAME, and WonderSwan documentation references. Identified and fixed two additional pipeline bugs not covered by the initial VTOTAL patch.

## Root Causes Found

1. **Low-VTOTAL frame-end off-by-one**
	- Nexen low-VTOTAL path ended at 145 scanlines instead of 144.
	- ares reference model ends at `max(144, vtotal + 1)` scanlines; for low VTOTAL this is exactly 144.
	- Fixed by changing frame end condition to:
		- `frameEndScanline = max(144, LastScanline + 1)`
		- end frame when `Scanline >= frameEndScanline`

2. **Low-VTOTAL post-frame white clear**
	- `SendFrame()` cleared rows for `LastScanline < 144`, which could overwrite wrapped/repeated image output and produce visible instability during effects.
	- Fixed by removing low-VTOTAL finalize whitening and preserving wrapped output.
	- Whitening remains only for sleep/LCD-off/power-off/`LastScanline == 255`.
