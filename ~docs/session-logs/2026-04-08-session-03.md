# Session Log — 2026-04-08 Session 03

## Summary

Fixed Shift+F1 Save State Picker crash (AccessViolationException) caused by concurrent native video filter access and unsafe pause byte reading.

## Issue

- **#1186** — Shift+F1 Save State Picker crashes with AccessViolationException in GetSaveStatePreview

## Root Cause

`StateGridEntry.Init()` spawned multiple `Task.Run()` calls that each invoked `EmuApi.GetSaveStatePreview()` concurrently. The C++ `GetSaveStatePreview` function uses `_emu->GetVideoFilter(true)` which is not thread-safe — concurrent access corrupted memory causing SEH exception 0xc0000005. The managed `catch { }` block cannot catch `AccessViolationException` in .NET 5+, so the process terminated.

Secondary issue: `GetSaveStateList()` read the last byte of every save state file as a pause flag without checking the file format version. Only v5+ saves have a pause byte appended.

## Fixes Applied

1. **Serialized preview loading** — Added `SemaphoreSlim` to `StateGridEntry` to serialize native `GetSaveStatePreview` calls (one at a time)
2. **Format version validation** — `GetSaveStateList()` now reads the MSS header and checks `fileFormatVersion >= 5` before reading the pause byte
3. **PNG buffer bounds check** — `GetSaveStatePreview()` now validates PNG output size against buffer capacity before `memcpy`
