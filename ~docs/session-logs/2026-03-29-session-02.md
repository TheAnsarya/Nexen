## Additional Work: #1076 WonderSwan FF2 Video Instability

### Research Summary

- Confirmed #1076 report details and maintainer request for savestate/hash metadata.
- Searched upstream SourMesen/Mesen2 issues and general web reports; did not find a confirmed existing upstream issue for this exact symptom.
- Compared local [Core/WS/WsPpu.cpp](Core/WS/WsPpu.cpp) behavior against upstream Mesen2 WS PPU logic and cross-checked ares WonderSwan timing behavior.

### Root-Cause Hypothesis

- Nexen was resetting frame progression based directly on `LastScanline` when it was below 144.
- This can shorten frame timing and suppress normal line-144 vblank cadence.
- Reference behavior repeats visible scanlines for low VTOTAL values until line 143, then enters vblank at the fixed point.

### Code Changes Applied

- Updated [Core/WS/WsPpu.cpp](Core/WS/WsPpu.cpp) to repeat rendered scanline input when `LastScanline + 1 < 144`.
- Updated frame wrap condition to end frame at `max(144, LastScanline + 1)` instead of ending immediately at `LastScanline`.
- Removed low-VTOTAL blanking path in `SendFrame` that cleared lower lines; this now preserves repeated-line output behavior.
