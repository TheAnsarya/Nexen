# Session Log ‚Äî 2026-04-09 Session 01

## Summary

Fixed three related crash bugs reported in issues #1176, #1194, and #1039 that prevented Nexen from starting on fresh installs in portable mode (Windows and Linux). Also fixed the Linux AoT `TaskCanceledException` crash on exit (#1195).

## Issues Addressed

### #1176 + #1194 ‚Äî Windows: Nexen won't start on fresh install

**Root cause:** `DependencyHelper.ExtractNativeDependencies()` unconditionally skipped extracting `.dll` files when the destination was the app's base directory (portable mode). On fresh installs, the native libraries (NexenCore.dll, libSkiaSharp.dll, libHarfBuzzSharp.dll) were never extracted from the embedded zip.

**Fix:** Only skip extraction when the file already exists at the destination, preserving the mmap safety guard for loaded libraries while allowing fresh installs to work.

### #1039 ‚Äî Linux: NexenCore.so missing in portable mode + FormatException on exit

**Root cause 1:** Same as #1176/#1194 ‚Äî `.so` files skipped in portable mode even on fresh install.

**Root cause 2:** `RomHashInfo.Crc32Value` used `uint.Parse()` on the CRC32 hex string. When the ROM was unloaded during exit, `ComputeRomHashes()` returned `RomHashInfo.Empty` (empty strings), causing `FormatException`.

**Fixes:**

- Same DependencyHelper fix as above
- Changed `Crc32Value` to use `uint.TryParse()`, returning 0 for empty/invalid strings

### Diagnostic improvement

- `DllImportResolver` error message now includes both AppBase and HomeFolder paths searched

### #1195 ‚Äî Linux AoT: TaskCanceledException crash on exit

**Root cause:** After `StartWithClassicDesktopLifetime` returns, Avalonia's dispatcher is disposed. `Tmds.DBus.Protocol` (transitive dependency from Avalonia on Linux) has a background message handler that calls `Dispatcher.Send()` during its own cleanup, throwing `TaskCanceledException` on a background thread. This was unhandled, causing the runtime to abort with a core dump.

**Fix (two layers):**

- Proactive: On Linux, `Environment.Exit(0)` after normal shutdown logging prevents the race entirely
- Defensive: `UnhandledException` handler detects `TaskCanceledException` originating from `Tmds.DBus` and exits cleanly instead of logging as fatal
