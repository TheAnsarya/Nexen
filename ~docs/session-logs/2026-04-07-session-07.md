## Root Cause

The C++ `SaveStateManager::LoadState()` correctly saves a pause byte (format v5+) and restores it on load by calling `Emulator::Pause()` or `Emulator::Resume()`. However, the C# `RecentGameInfo.Load()` method in `RecentGamesViewModel.cs` unconditionally called `EmuApi.Resume()` after every load operation, overriding the restored pause state from the save data.

Three load paths were affected:

- Timestamped save file load
- Slot-based state load
- Recent game load

## Fix

Removed all three unconditional `EmuApi.Resume()` calls from `RecentGameInfo.Load()`. The C++ layer already handles pause/resume restoration correctly. For the save path (slot-based save), the `OnCloseClick` handler in `StateGrid` already resumes via `NeedResume` when the dialog closes.

**File:** `UI/ViewModels/RecentGamesViewModel.cs`
