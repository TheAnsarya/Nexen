# 2026-05-15 Session 05

## Summary

Follow-up CI patch after run `25915375155` narrowed failures to Windows InteropDLL include resolution and Linux gcc warning gate.

## Failures Diagnosed

- Windows (`AoT` and non-`AoT`) failed in `Build C++ core (InteropDLL)` with missing includes:
	- `miniz/miniz.h`
	- `magic_enum/magic_enum.hpp`
- Linux `gcc` x64 failed warning regression gate with `delta=114`.

## Root Cause

- Windows MSBuild parameter used a malformed quoted path:
	- `-p:VcpkgInstalledDir="...\installed\"`
	- trailing backslash escaped the closing quote and broke property parsing.
- Linux gcc baseline was still tuned below current known steady-state warning count.

## Changes Made

- Updated `.github/workflows/build.yml`:
	- Fixed Windows `VcpkgInstalledDir` argument to `...\vcpkg\installed` (no trailing escaped quote issue).
	- Applied Linux warning baseline override for all `gcc` matrix legs (`4192`).
