2. **#1224 тАФ The sound goes high-pitched when using fast forward**: Root cause was `SoundMixer.cpp` applying `HermiteResampler` pitch adjustment for ALL non-100% speeds including fast-forward. Fixed by changing condition from `emulationSpeed != 100` to `emulationSpeed < 100`.

## Technical Details

### #1223 тАФ Property Registration Conflict

`KeyBindingButton` registers `StyledProperty<UInt16>` named "KeyBinding". `MultiKeyBindingButton` tried to register `StyledProperty<KeyCombination>` also named "KeyBinding" but under the same owner `KeyBindingButton` (wrong generic parameter). Avalonia throws `InvalidOperationException` for same-name different-type properties on the same owner.

Exception flow: Preferences тЖТ Shortcuts tab тЖТ `MultiKeyBindingButton` static init тЖТ property conflict тЖТ global exception handler тЖТ error dialog.

### #1224 тАФ Audio Pitch Adjustment

The `HermiteResampler` is an interpolation-based sample rate converter that inherently changes pitch. The pitch adjustment formula `SetSampleRates(targetRate, targetRate * 100.0 / emulationSpeed)` at 3x speed creates a 3:1 decimation ratio, compressing audio so 440Hz becomes 1320Hz.

Upstream Mesen2 uses `if(emulationSpeed > 0 && emulationSpeed < 100)` тАФ only adjusting for slow speeds. At fast speeds, audio plays at normal pitch with excess samples dropped by the audio device buffer. Nexen's existing `emulationSpeed == 0` guard (unlimited speed skips audio entirely) makes the `> 0` check redundant, so `emulationSpeed < 100` is sufficient.
