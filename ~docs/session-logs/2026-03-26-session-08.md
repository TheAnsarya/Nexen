### Multi-Instance Infrastructure Research

- `IConsole.h`: Abstract interface, single-threaded model, no linking methods
- `Emulator` class owns exactly ONE `safe_ptr<IConsole>` — no multi-instance support
- GB serial: loopback-only, WS serial: stub, Netplay: input-sync only
- **Conclusion:** Zero cross-instance infrastructure exists

### Architecture Decision

Evaluated 4 options:

| Option | Description | Decision |
|--------|-------------|----------|
| A | Multi-console Emulator | Rejected — too invasive |
| B | Shared memory IPC | Rejected — overengineered |
| C | Test-only harness | Rejected — no runtime usage |
| **D** | **Cable abstraction + test harness** | **Chosen** — minimal, correct, extensible |

### Design Details

- `ComLynxCable` connects multiple `LynxMikey*` instances
- `Broadcast()` calls `ComLynxRxData()` (back-insert) on all connected units except sender
- Sender self-loopback via `ComLynxTxLoopback()` (front-insert) happens first — preserving §7.2 priority
- `_comLynxCable` defaults to `nullptr`; `[[unlikely]]` branch hint ensures zero overhead in single-player
- Non-copyable, non-movable, single-threaded design

## Benchmark Results

| Benchmark | Mean Time | Notes |
|-----------|-----------|-------|
| NullCheck (no cable) | ~8.7ns | Near-zero single-player overhead |
| Broadcast 2 units | ~7.0ns | 2-player cost |
| Broadcast 4 units | ~8.9ns | 4-player cost |
| Broadcast 8 units | ~14.5ns | Stress test, linear scaling |
| Full TX path 2 units | ~68ns | End-to-end with TickUart |
| Full TX path 4 units | ~128ns | Linear scaling confirmed |
| TX loopback only | ~3.8ns | Single-player baseline |
| Loopback + broadcast | ~6.7ns | ~3ns cable overhead |
| Burst 16B 2 units | ~42ns | Protocol packet simulation |
| Burst 16B 4 units | ~77ns | Scales linearly |
| TickIdle with cable | ~6.9ns | Idle path unchanged |

**Key finding:** Cable broadcast adds ~3ns per TX byte for 2-player — negligible at 960 bytes/sec (9600 baud).
