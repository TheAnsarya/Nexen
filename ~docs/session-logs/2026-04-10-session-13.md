# Session Log — 2026-04-10 Session 13

## Summary

Fixed debugger window close crash caused by race condition where pending `Dispatcher.UIThread.Post` callbacks accessed disposed view models during window teardown.

## Issue

- **#1222** — fix: debugger window close crash, menu unusability, and broken keyboard shortcuts

## Root Cause

When closing the last debug window, `DebugWindowManager.CloseDebugWindow` calls `Dispatcher.UIThread.RunJobs()` inside a write lock. This processes pending callbacks from `ProcessNotification` that were enqueued before the window was removed from `_openedWindows`. These callbacks access `_model` methods (`UpdateDebugger`, `PartialRefresh`, `ProcessResumeEvent`) on already-disposed `DebuggerWindowViewModel`, causing crashes. The menu unusability and keyboard shortcut issues are likely secondary symptoms of the same crash/state corruption.
